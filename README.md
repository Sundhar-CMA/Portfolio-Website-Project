# Finance & Analytics Portfolio Website

## Project Overview

This repository documents is the creation of my professional portfolio website, designed to showcase my experience, skills, projects, certifications, volunteering activities, and continuous learning journey.

The website was built using Lovable AI as the development platform, supported by ChatGPT for prompt refinement and project planning. While AI assisted with implementation, the overall vision, structure, content, and user experience decisions were driven by me.

The project represents an exploration of how AI can be used as a productivity and development partner while maintaining human control over design decisions, content quality, and professional branding.

## Key Challenges

### 1. Preventing Unintended Website Changes

One of the biggest risks when working with AI-generated updates is accidental modification of unrelated sections.

Challenge:

* A prompt intended for one section could unexpectedly affect another.
* Styling, navigation, layouts, or previously completed sections could be altered.

Solution:

* Adopted a highly controlled prompting approach.
* Explicitly instructed the AI to modify only the target section.
* Repeated constraints such as:

"Do not modify any other sections."

"Maintain existing design, colors, typography, spacing, responsiveness, and layout."

* Implemented updates one section at a time rather than making multiple changes simultaneously.

This significantly reduced hallucinations and unintended modifications.

---

### 2. Limited AI Credits and Resource Optimization

As a free-tier Lovable AI user, I had a limited number of daily credits available.

Challenge:

* Every unnecessary prompt consumed valuable credits.
* Large prompts increased the risk of unintended changes.
* Repeated corrections could quickly exhaust available credits.

Solution:

* Planned changes before executing them.
* Broke large requirements into smaller implementation phases.
* Prioritized high-impact updates first.
* Used ChatGPT to draft and optimize prompts before submitting them to Lovable AI.
* Minimized trial-and-error by improving requirement clarity upfront.


---

### 3. Translating Vision into Clear Requirements

AI can only execute instructions that are clearly communicated.

Challenge:

* I knew how I wanted the website to behave.
* Converting those ideas into structured instructions required careful thought.

Examples:

* Certificate pop-up viewers
* Certificate carousels
* Navigation flow
* Section hierarchy
* Responsive behavior

Solution:

* Defined expected user experience before writing prompts.
* Created rough requirements manually.
* Refined those requirements using ChatGPT.
* Converted concepts into detailed implementation instructions.

---

### 4. Balancing Simplicity and Functionality

Challenge:

* Avoid feature overload.
* Maintain professional appearance.
* Keep navigation intuitive.

Examples:

* Deciding against adding a photo gallery.
* Keeping certifications organized through modal viewers.
* Using GitHub links instead of embedding large project files.
* Maintaining a clean finance-focused design.

Solution:

* Evaluated every feature against a simple question:

"Does this improve the visitor experience?"

If not, it was removed.

## Prompt Engineering Framework

A structured framework was developed during the project to maximize accuracy and reduce AI errors.

### Step 1 – Define Objective

Clearly identify:

* What needs to be changed
* Why it needs to be changed
* Expected outcome

Example:

"Add a certification modal for the US CMA certificate."

---

### Step 2 – Protect Existing Work

Before any modification:

Specify constraints:

* Do not redesign website
* Do not change colors
* Do not modify layout
* Do not change typography
* Maintain responsiveness

This reduced unintended modifications.

---

### Step 3 – Isolate the Target Section

Focus on one section only.

Examples:

* Certifications
* Volunteering
* Projects
* Navigation

This reduced hallucinations and improved accuracy.

---

### Step 4 – Define User Experience

Describe exactly how users should interact with the feature.

Example:

When the user clicks "View Certificate":

* Open modal
* Do not leave website
* Display certificate
* Maintain background visibility
* Include close button

---

### Step 5 – Define Technical Constraints

Specify:

* Modal size
* Carousel behavior
* Responsiveness
* Accessibility requirements
* Navigation behavior

---

### Step 6 – Validate Before Execution

Before submitting prompts:

* Review requirements.
* Simplify wording.
* Remove ambiguity.
* Ensure no conflicting instructions exist.
